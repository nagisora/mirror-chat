# MirrorChat Chrome拡張 利用手順

[English](CHROME_EXTENSION_USAGE.md) | [日本語](CHROME_EXTENSION_USAGE_ja.md)

## 1. インストール

1. `ai-prompt-broadcaster` フォルダを任意の場所に配置します。
2. Chrome で `chrome://extensions/` を開きます。
3. 右上のデベロッパーモードを ON にします。
4. パッケージ化されていない拡張機能を読み込む をクリックします。
5. `ai-prompt-broadcaster` フォルダを選択します。

## 2. Obsidian の準備

1. Obsidian を起動し、Vault を開きます。
2. 設定のコミュニティプラグインから、必要に応じて 制限されたモード をオフにします。
3. Local REST API を検索してインストールします。
4. プラグインを有効化し、設定画面で以下を確認します。
   - ポート番号。一般的には HTTPS が 27124、HTTP が 27123
   - 必要に応じて API トークン

## 3. 拡張の設定

1. 拡張アイコンを右クリックして オプション を開きます。
2. Obsidian Local REST API ベース URL を設定します。例: `https://127.0.0.1:27124/`
3. 必要に応じて Obsidian の API トークンを入力します。
4. 保存ルートパスを設定します。例: `200-AI Research`
5. digest を使う場合は、回答保存後の非同期 digest 生成を有効化し、OpenRouter API キーを設定します。
6. 保存 をクリックします。
7. 最新の free モデル候補が必要な場合は free 候補を更新 を実行します。

## 4. 利用手順

1. 可能なら事前に ChatGPT、Claude、Gemini、Grok のいずれかへブラウザでログインしておきます。
2. 拡張アイコンをクリックします。
3. 質問を入力して 送信 をクリックします。
4. 拡張が4つの AI サイトを順次開き、自動でプロンプトを送信します。
5. 回答取得後、Obsidian Vault の `保存ルートパス/YYYYMMDD-連番-質問の先頭20文字/` 配下に `01-質問文抜粋.md` のようなファイルが作成されます。
6. 各質問ファイルには `## 質問`、`## まとめ`、各AI回答セクションが含まれます。digest 有効時は、最初に待機状態の文言が入り、あとから OpenRouter の digest に置き換わります。
7. digest 生成に失敗しても raw 回答は保存され、まとめセクションに失敗状態が反映されます。

## 5. トラブルシューティング

### 保存に失敗する

- Obsidian が起動しているか確認します。
- Local REST API プラグインが有効か確認します。
- ベース URL と API トークンが正しいか確認します。
- 失敗項目は拡張機能から再試行できます。

### 特定の AI で回答が取得できない

- そのサービスにログインしているか確認します。
- provider 側 UI 変更の可能性があるため、Options 画面で DOM セレクタを調整します。
- ブラウザの開発者ツールで要素を確認し、適切なセレクタを設定します。
